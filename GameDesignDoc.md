# Neuro Mines（神经扫雷）— 游戏设计文档 v3

> 核心定位：神经刺激型反应游戏，不是传统扫雷。
> 设计哲学：**界面要消失，体验要放大。**
> 当前阶段：离线单机版，多人测试稳定后再加联网/社交。

---

## 一、产品定位

| 维度 | 定义 |
|------|------|
| 类型 | 超休闲 + 轻竞技 |
| 单局时长 | 60秒（优柔寡断模式 180秒） |
| 核心场景 | 等电梯、刷视频间隙、通勤碎片 |
| 卖点 | 60秒一局（短）、连击+修饰器（爽）、红蓝雷+勇气时刻（刺激） |
| 当前技术栈 | HTML5 Canvas + CSS + 原生 JS（零依赖） |
| 目标平台 | 移动端优先 → PWA / App（买断制） → 可选微信小程序 |

## 二、已实现功能清单（v1 当前状态）

### 核心玩法
- [x] 种子驱动棋盘生成（Mulberry32 PRNG）
- [x] 约束传播求解器（基础规则 + 子集约束规则，距离≤2 邻居对）
- [x] BFS 连锁翻开 + 波纹动画
- [x] 3 条命容错（踩红雷扣命）
- [x] 所有关卡统一 60 秒基础时间
- [x] Combo 系统（×1/×2/×3 倍率，≥5连×2，≥10连×3）
- [x] Chord 操作（点击已翻开数字格自动翻开周围）
- [x] 标旗模式（按钮切换 + 右键）
- [x] 关卡递进系统（6×6 → 7×7 → 8×8，雷密度 17%~38%）
- [x] localStorage 最高纪录

### 红蓝雷系统（L6 起）
- [x] 🔴 红雷：扣命 + 扣50分
- [x] 🔵 蓝雷：不扣命，扣剩余时间（分阶段递增）
  - L6–10：扣1/4 剩余时间
  - L11–17：扣1/3 剩余时间
  - L18+：扣1/2 剩余时间
- [x] L6-10 蓝雷占30%，L11+ 占50%
- [x] 蓝色爆炸特效 + 独立音效

### 修饰器系统（每关开局选择）
- [x] 🤔 优柔寡断：时间+200%（180秒），得分 ×0.5 — 始终展示
- [x] 🔥 孤注一掷：生命=1，得分 ×1.6
- [x] ⏱️ 争分夺秒：时间-20%（48秒），得分 ×1.3
- [x] 🚫 盲扫大师：无法标旗，得分 ×1.3
- [x] 🎰 赌徒本能：半可解棋盘（40%–70%可推理），得分 ×1.3
- [x] 可叠加，倍率相乘
- [x] 优柔寡断始终展示 + 从其余4个中随机抽3个

### 时间奖励
- [x] 链式奖励：翻开 ≥5 格 → +2s
- [x] 速推系统（按关卡阶段收紧）：
  - L1–7：4步/10s +4s，6步/8s +6s
  - L8–14：4步/8s +3s，6步/7s +5s
  - L15–22：4步/7s +3s，6步/5s +5s
  - L23+：4步/6s +2s，6步/4s +4s
- [x] 所有时间加成不超过关卡上限

### 勇气时刻
- [x] 条件：逻辑求解器判定无安全步（hasSafeMove() = false）
- [x] 分数奖励：本关得分 × 当前进度 × 衰减系数（连续勇气 0.5^n，正常翻开重置）
- [x] 生存奖励（进度 ≥ 60%）：+1 生命（满血则本关得分 +25%）
- [x] 生存奖励（进度 < 60%）：回复时间 = 时间上限 × 进度
- [x] 分级特效：进度 ≥ 40% 完整特效（金色脉冲+骰子+碎纸屑），< 40% 仅飘分轻反馈
- [x] 3 档动画强度：普通（轻闪）、中等（2s 叠层）、强烈（2.5s + 震动）
- [x] 6 音上行琶音 + 闪光和弦

### 无伤通关
- [x] 一关内零踩雷 → 本关得分 +10%
- [x] 结算画面显示 🛡️无伤加分

### 其他
- [x] 暂停/继续（遮罩防偷看）
- [x] 超时全屏爆炸（雷依次翻出 + 红闪 + 强震）
- [x] 通关回复1❤️ + 胜利翻金动画
- [x] Web Audio 程序合成全套音效

### Sprint 1：触控手感基础（Day 1 下半天 ~ Day 2）

> "点了有反应" → "点了很舒服"。这一步决定玩家会不会玩第二局。

| # | 任务 | 说明 | 预计 |
|---|------|------|------|
| 1.1 | 触摸区域优化 | 格子点击判定扩大 2px 容差，防止边缘误触 | 30min |
| 1.2 | 按压态反馈 | touchstart 时格子缩小至 0.9 + 变色，touchend 恢复 | 1h |
| 1.3 | 长按标旗 | 300ms 长按 = 标旗，带进度指示（圆环填充）| 1.5h |
| 1.4 | Vibration API | 全场景接入（见下方震动表），iOS 不支持时静默降级 | 1h |
| 1.5 | 防误触处理 | 滑动距离 > 10px 取消点击，双指缩放禁止 | 30min |
| 1.6 | 安全区适配 | `env(safe-area-inset-*)` 处理刘海屏/底部横条 | 30min |

**震动方案（Vibration API）：**

| 场景 | 震动 | 代码 |
|------|------|------|
| 翻格 | 微震 15ms | `navigator.vibrate(15)` |
| 连锁翻开 | 每波 10ms | `navigator.vibrate([10, 30, 10, 30...])` |
| 踩红雷 | 中震 50ms | `navigator.vibrate(50)` |
| 踩蓝雷 | 双短震 | `navigator.vibrate([20, 40, 20])` |
| 勇气时刻 | 强脉冲 ×3 | `navigator.vibrate([100, 80, 100, 80, 100])` |
| 通关 | 升调三连 | `navigator.vibrate([30, 50, 50, 50, 80])` |
| 高 Combo（10+） | 节奏震 | `navigator.vibrate([15, 30, 15, 30, 40])` |

**验收标准**：单手拇指可以流畅完成一整局，长按标旗自然，不会误点

### Sprint 3：音效升级（Day 4 ~ Day 5）

> 用声音塑造节奏感。不加 BGM — 扫雷的紧张感来自安静。

| # | 任务 | 说明 | 预计 |
|---|------|------|------|
| 3.1 | 音效素材制作 | 用 Web Audio 程序合成 5 个核心音效，导出 .mp3 备用 | 2h |
| 3.2 | 翻格音效升级 | "嗒"声 + pitch 随 combo 升高（C4→E4→G4→C5） | 1h |
| 3.3 | 爆炸音效分化 | 红雷：低频轰鸣 + 碎裂；蓝雷：电子故障 + 时钟倒转 | 1h |
| 3.4 | 连锁音阶 | 翻开多格时 do-re-mi-fa 上行音阶，宫调式 | 1h |
| 3.5 | 勇气/通关音效精修 | 勇气：6 音上行 + 和弦尾音；通关：完整 2 秒旋律 | 1h |
| 3.6 | 修饰器选择音效 | 选中：金属锁扣"咔"；取消：轻弹回"嘀" | 30min |
| 3.7 | 音量控制 | 添加静音按钮，记住用户偏好到 localStorage | 30min |

**音效设计原则：**
- 所有音效 < 0.5 秒（勇气/通关除外）
- 总音频资源 < 50KB（优先程序合成，文件兜底）
- 高 Combo 时 pitch 自然升高，给玩家"越来越快"的心理节奏
- 不加 BGM — 安静中突然响起的翻格声就是最好的节奏

**验收标准**：闭眼听声音，能分辨"安全翻开/踩红雷/踩蓝雷/连锁/通关"

## 七、移动端发布方案 — 从零到上架

> 适用于没做过手机 App 的开发者。当前项目是纯 HTML/CSS/JS，有三条路。

### 方案对比

| 方案 | 说明 | 上手难度 | 到 App Store | 适合阶段 |
|------|------|----------|-------------|----------|
| **A. PWA** | 网页直接"添加到主屏幕" | ★☆☆ | 不上架 | 朋友测试、快速迭代 |
| **B. Capacitor 套壳** | 网页打包成原生 App | ★★☆ | 可上架 | 正式发布 |
| **C. 微信小程序** | 移植到小程序框架 | ★★☆ | 微信内发布 | 国内传播 |

### 方案 A：PWA（推荐先做，零成本验证）

**什么是 PWA**：把网页变成"像 App 一样"的东西。用户在浏览器里打开网页 → 点"添加到主屏幕" → 桌面出现图标 → 打开后全屏运行，看不到浏览器地址栏。

**需要做的事**：

```
1. 创建 manifest.json（App 名称、图标、配色、启动方式）
2. 创建 service-worker.js（缓存静态资源，实现离线访问）
3. 在 index.html 里引入 manifest
4. HTTPS 部署（GitHub Pages 免费）
```

**优点**：改动量极小（<1小时），不需要任何新工具，GitHub Pages 免费托管，朋友扫码即可玩。
**缺点**：iOS 上 PWA 限制较多（无推送、存储可能被清）、不在 App Store。

**发布流程**：
```
代码推送 GitHub → 开启 GitHub Pages → 拿到 https://xxx.github.io/ScanBoomBoom/
→ 生成二维码 → 朋友扫码 → Safari/Chrome 打开 → "添加到主屏幕"
```

### 方案 B：Capacitor 套壳打包成 App（正式发布用）

**什么是 Capacitor**：Ionic 团队出的工具，把现有网页直接包进一个原生 App 壳里。你的 HTML/CSS/JS 代码**不用改**，它帮你生成 iOS/Android 项目。

**工具栈**：

```
- Node.js 18+（你应该已经有）
- @capacitor/core + @capacitor/cli（npm 安装）
- Android Studio（构建 Android APK）
- Xcode（构建 iOS IPA，需要 Mac）
- Apple 开发者账号（$99/年，上架 App Store 需要）
- Google Play 开发者账号（$25 一次性，上架 Play Store）
```

**具体流程**：

```
Step 1: 初始化
  npm init -y
  npm install @capacitor/core @capacitor/cli
  npx cap init "Neuro Mines" "com.yourname.neuromines"
  → 生成 capacitor.config.ts

Step 2: 指定网页目录
  配置 webDir 指向你的项目根目录（有 index.html 的那个）

Step 3: 添加平台
  npm install @capacitor/android @capacitor/ios
  npx cap add android
  npx cap add ios
  → 生成 android/ 和 ios/ 目录

Step 4: 同步代码
  npx cap sync
  → 把你的网页文件复制进原生项目

Step 5: 打开原生 IDE
  npx cap open android  → 打开 Android Studio
  npx cap open ios      → 打开 Xcode

Step 6: 构建 & 测试
  Android Studio: Build → Run（连手机或模拟器）
  Xcode: Product → Run（连 iPhone 或模拟器）

Step 7: 接入原生能力
  npm install @capacitor/haptics   → 震动 API
  npm install @capacitor/status-bar → 状态栏控制
  npm install @capacitor/splash-screen → 启动屏
  → 在 JS 里直接 import 调用

Step 8: 上架
  Android: Build → Generate Signed APK → 上传 Google Play Console
  iOS: Archive → 上传 App Store Connect → 提交审核
```

**优点**：现有代码几乎不改、原生震动/状态栏/启动屏、可上架应用商店。
**缺点**：需要 Mac 才能出 iOS、App Store 审核周期 1-3 天。

### 方案 C：微信小程序（国内传播最佳）

**差异**：小程序有自己的 API，不能直接用 DOM。但 Canvas API 基本一致。

**需要改的**：

| 模块 | 改动量 |
|------|--------|
| Canvas 渲染（renderer.js） | 小改（wx.createCanvas） |
| 触摸事件 | 小改（touchstart/end 换成小程序 API） |
| HTML 结构 | 重写为 WXML |
| CSS | 重写为 WXSS（基本兼容） |
| localStorage | 换成 wx.setStorageSync |
| Audio | 换成 wx.createInnerAudioContext |
| 分享 | 原生 wx.shareAppMessage（巨大优势）|

**工具栈**：
```
- 微信开发者工具（免费下载）
- 微信小程序账号（免费注册，个人可发布）
- 无需 Mac，无需开发者付费账号
```

**优点**：微信内直接分享转发（传播系数最高）、无需安装、审核快（1天）。
**缺点**：需要重写 UI 层、受微信平台限制。

### 推荐路线

```
现在 → PWA 部署（1小时搞定，朋友测试）
     → 收集反馈、打磨手感（第二阶段 A/B）
     → Capacitor 打包 Android APK（给更多人测）
     → 决定是否上架 / 是否做小程序
```

## 九、核心设计原则

1. **单局 60秒**：地图小、决策密度高、3秒进游戏
2. **高频正反馈**：连锁波纹、Combo跳字、勇气特效
3. **低认知负担**：半随机+半技巧，不强依赖复杂推理
4. **错误不致命**：3条命容错 + 蓝雷只扣时间 + 勇气奖励救场
5. **风险/回报可选**：5种修饰器让玩家自己选择压力等级
6. **静态极简，操作爆发**：安静时干净，点击时爽炸
