# Neuro Mines（神经扫雷）— 游戏设计文档 v3

> 核心定位：神经刺激型反应游戏，不是传统扫雷。
> 设计哲学：**界面要消失，体验要放大。**
> 当前阶段：离线单机版，多人测试稳定后再加联网/社交。

---

## 一、产品定位

| 维度 | 定义 |
|------|------|
| 类型 | 超休闲 + 轻竞技 |
| 单局时长 | 60秒（优柔寡断模式 180秒） |
| 核心场景 | 等电梯、刷视频间隙、通勤碎片 |
| 卖点 | 60秒一局（短）、连击+修饰器（爽）、红蓝雷+勇气时刻（刺激） |
| 当前技术栈 | HTML5 Canvas + CSS + 原生 JS（零依赖） |
| 目标平台 | 移动端优先 → PWA / App（买断制） → 可选微信小程序 |

## 二、已实现功能清单（v1 当前状态）

### 核心玩法
- [x] 种子驱动棋盘生成（Mulberry32 PRNG）
- [x] 约束传播求解器（基础规则 + 子集约束规则，距离≤2 邻居对）
- [x] BFS 连锁翻开 + 波纹动画
- [x] 3 条命容错（踩红雷扣命）
- [x] 所有关卡统一 60 秒基础时间
- [x] Combo 系统（×1/×2/×3 倍率，≥5连×2，≥10连×3）
- [x] Chord 操作（点击已翻开数字格自动翻开周围）
- [x] 标旗模式（按钮切换 + 右键）
- [x] 关卡递进系统（6×6 → 7×7 → 8×8，雷密度 17%~38%）
- [x] localStorage 最高纪录

### 红蓝雷系统（L6 起）
- [x] 🔴 红雷：扣命 + 扣50分
- [x] 🔵 蓝雷：不扣命，扣剩余时间（分阶段递增）
  - L6–10：扣1/4 剩余时间
  - L11–17：扣1/3 剩余时间
  - L18+：扣1/2 剩余时间
- [x] L6-10 蓝雷占30%，L11+ 占50%
- [x] 蓝色爆炸特效 + 独立音效

### 修饰器系统（每关开局选择）
- [x] 🤔 优柔寡断：时间+200%（180秒），得分 ×0.5 — 始终展示
- [x] 🔥 孤注一掷：生命=1，得分 ×1.6
- [x] ⏱️ 争分夺秒：时间-20%（48秒），得分 ×1.3
- [x] 🚫 盲扫大师：无法标旗，得分 ×1.3
- [x] 🎰 赌徒本能：半可解棋盘（40%–70%可推理），得分 ×1.3
- [x] 可叠加，倍率相乘
- [x] 优柔寡断始终展示 + 从其余4个中随机抽3个

### 时间奖励
- [x] 链式奖励：翻开 ≥5 格 → +2s
- [x] 速推系统（按关卡阶段收紧）：
  - L1–7：4步/10s +4s，6步/8s +6s
  - L8–14：4步/8s +3s，6步/7s +5s
  - L15–22：4步/7s +3s，6步/5s +5s
  - L23+：4步/6s +2s，6步/4s +4s
- [x] 所有时间加成不超过关卡上限

### 勇气时刻
- [x] 条件：逻辑求解器判定无安全步（hasSafeMove() = false）
- [x] 分数奖励：本关得分 × 当前进度 × 衰减系数（连续勇气 0.5^n，正常翻开重置）
- [x] 生存奖励（进度 ≥ 60%）：+1 生命（满血则本关得分 +25%）
- [x] 生存奖励（进度 < 60%）：回复时间 = 时间上限 × 进度
- [x] 分级特效：进度 ≥ 40% 完整特效（金色脉冲+骰子+碎纸屑），< 40% 仅飘分轻反馈
- [x] 3 档动画强度：普通（轻闪）、中等（2s 叠层）、强烈（2.5s + 震动）
- [x] 6 音上行琶音 + 闪光和弦

### 无伤通关
- [x] 一关内零踩雷 → 本关得分 +10%
- [x] 结算画面显示 🛡️无伤加分

### 其他
- [x] 暂停/继续（遮罩防偷看）
- [x] 超时全屏爆炸（雷依次翻出 + 红闪 + 强震）
- [x] 通关回复1❤️ + 胜利翻金动画
- [x] Web Audio 程序合成全套音效

|------|------|------|------|------|----------|----------|

### Sprint 1：触控手感基础（Day 1 下半天 ~ Day 2）

> "点了有反应" → "点了很舒服"。这一步决定玩家会不会玩第二局。

| # | 任务 | 说明 | 预计 |
|---|------|------|------|
| 1.1 | 触摸区域优化 | 格子点击判定扩大 2px 容差，防止边缘误触 | 30min |
| 1.2 | 按压态反馈 | touchstart 时格子缩小至 0.9 + 变色，touchend 恢复 | 1h |
| 1.3 | 长按标旗 | 300ms 长按 = 标旗，带进度指示（圆环填充）| 1.5h |
| 1.4 | Vibration API | 全场景接入（见下方震动表），iOS 不支持时静默降级 | 1h |
| 1.5 | 防误触处理 | 滑动距离 > 10px 取消点击，双指缩放禁止 | 30min |
| 1.6 | 安全区适配 | `env(safe-area-inset-*)` 处理刘海屏/底部横条 | 30min |

**震动方案（Vibration API）：**

| 场景 | 震动 | 代码 |
|------|------|------|
| 翻格 | 微震 15ms | `navigator.vibrate(15)` |
| 连锁翻开 | 每波 10ms | `navigator.vibrate([10, 30, 10, 30...])` |
| 踩红雷 | 中震 50ms | `navigator.vibrate(50)` |
| 踩蓝雷 | 双短震 | `navigator.vibrate([20, 40, 20])` |
| 勇气时刻 | 强脉冲 ×3 | `navigator.vibrate([100, 80, 100, 80, 100])` |
| 通关 | 升调三连 | `navigator.vibrate([30, 50, 50, 50, 80])` |
| 高 Combo（10+） | 节奏震 | `navigator.vibrate([15, 30, 15, 30, 40])` |

**验收标准**：单手拇指可以流畅完成一整局，长按标旗自然，不会误点

### Sprint 3：音效升级（Day 4 ~ Day 5）

> 用声音塑造节奏感。不加 BGM — 扫雷的紧张感来自安静。

| # | 任务 | 说明 | 预计 |
|---|------|------|------|
| 3.1 | 音效素材制作 | 用 Web Audio 程序合成 5 个核心音效，导出 .mp3 备用 | 2h |
| 3.2 | 翻格音效升级 | "嗒"声 + pitch 随 combo 升高（C4→E4→G4→C5） | 1h |
| 3.3 | 爆炸音效分化 | 红雷：低频轰鸣 + 碎裂；蓝雷：电子故障 + 时钟倒转 | 1h |
| 3.4 | 连锁音阶 | 翻开多格时 do-re-mi-fa 上行音阶，宫调式 | 1h |
| 3.5 | 勇气/通关音效精修 | 勇气：6 音上行 + 和弦尾音；通关：完整 2 秒旋律 | 1h |
| 3.6 | 修饰器选择音效 | 选中：金属锁扣"咔"；取消：轻弹回"嘀" | 30min |
| 3.7 | 音量控制 | 添加静音按钮，记住用户偏好到 localStorage | 30min |

**音效设计原则：**
- 所有音效 < 0.5 秒（勇气/通关除外）
- 总音频资源 < 50KB（优先程序合成，文件兜底）
- 高 Combo 时 pitch 自然升高，给玩家"越来越快"的心理节奏
- 不加 BGM — 安静中突然响起的翻格声就是最好的节奏

**验收标准**：闭眼听声音，能分辨"安全翻开/踩红雷/踩蓝雷/连锁/通关"

#### 7.4.2 移植改动量

| 模块 | 改动 | 工作量 |

#### 7.4.3 广告变现策略

> 原则：**广告不能破坏游戏节奏**。玩家在高紧张度的 60 秒内绝对不插广告。

| 广告类型 | 触发时机 | 频率 | 预估 eCPM |
|----------|----------|------|-----------|
| **激励视频**（主力收入） | Game Over 后"看广告复活"（+1 命 + 恢复 15s） | 每局最多 1 次 | ¥30-80 |
| **激励视频** | 通关后"看广告双倍积分" | 每关可选 | ¥30-80 |
| **插屏广告** | 每 3 关结算后 | 非连续显示 | ¥10-30 |
| **Banner** | ❌ 不用 | — | — |

**广告设计原则**：
1. 激励视频 **必须是用户主动点击**，永远不强制
2. "看广告复活" 是核心变现点 — 玩家刚好在最紧张的时刻死掉，复活意愿最强
3. 插屏广告**只在自然中断点**出现（关卡结算画面），且不连续出现
4. 不用 Banner — 它会挤压游戏画面，收益还低

**收入预估**（保守/乐观）：

| 指标 | 保守 | 乐观 |

| DAU | 1,000 | 10,000 |
| 激励视频观看率 | 30% | 50% |
| 日均激励视频展示 | 300 次 | 5,000 次 |
| 激励视频 eCPM | ¥40 | ¥60 |
| 日收入 | ¥12 | ¥300 |
| 月收入 | ¥360 | ¥9,000 |

> 关键变量是 DAU。小程序冷启动靠社交裂变 → 分享功能必须做好（结算卡片 + 挑战链接）。

## 九、核心设计原则

1. **单局 60秒**：地图小、决策密度高、3秒进游戏
2. **高频正反馈**：连锁波纹、Combo跳字、勇气特效
3. **低认知负担**：半随机+半技巧，不强依赖复杂推理
4. **错误不致命**：3条命容错 + 蓝雷只扣时间 + 勇气奖励救场
5. **风险/回报可选**：5种修饰器让玩家自己选择压力等级
6. **静态极简，操作爆发**：安静时干净，点击时爽炸
