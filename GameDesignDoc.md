# Neuro Mines（神经扫雷）— 完整游戏设计方案 v2

> 核心定位：不是在做"扫雷"，而是在做一个**神经刺激型反应游戏**。
> 设计哲学：**界面要消失，体验要放大。**

---

## 一、产品定位

| 维度 | 定义 |
|------|------|
| 类型 | 超休闲 + 轻竞技 |
| 单局时长 | ≤ 60秒（硬指标） |
| 核心用户场景 | 等电梯、刷视频间隙、通勤碎片时间 |
| 卖点 | 60秒一局（短）、高频连击（爽）、迷雾紧张感（刺激）、好友同图比分（社交） |
| 技术栈 | Unity + UGUI + DOTween + 新Input System |
| 平台 | 移动端优先（iOS/Android），后续可出微信小程序 |

---

## 二、核心设计三原则

### 原则1：单局 ≤ 60秒
- 地图必须小（6×6 ~ 8×8），决策密度高
- 3秒进入游戏，失败后0.5秒可重开
- 无多余菜单，打开即玩

### 原则2：高频正反馈（替代推理成就感）
- 点开有连锁爆开动画
- 连续正确 → Combo奖励
- **错误不直接失败**（容错机制）

### 原则3：低认知负担
- 不强依赖复杂逻辑推理
- 半随机 + 半技巧
- 每局只改变1~2个变量（黄金法则）

## 三、核心玩法 — 极速连锁模式（Chain Mines）

### 3.1 基础规则改造

| 传统扫雷 | Neuro Mines |
|----------|-------------|
| 点到雷 = 直接死 | 点到雷 = 扣血（3条命） |
| 无时间限制 | 60秒倒计时 |
| 无连击 | 连续正确触发 Combo |
| 静态地图 | 种子驱动动态生成 |
| 所有数字立即可见 | 迷雾区域延迟显示（进阶） |
| 全局难度均匀 | 单局内难度递增（建立→加速→高压） |

### 3.2 六大核心机制

#### 机制1：连锁展开（爽点核心）
- 点击安全格 → 自动扩散（保留原版逻辑）
- **关键改造**：加入延迟传播动画，每格延迟30~50ms依次翻开
- 视觉效果类似"水波扩散"，制造连锁爽感
- 扩散过程中加入缩放+透明度渐变动效

#### 机制2：Combo系统（上瘾核心）
```
连续正确操作：
  1~4次  → +1 combo，正常计分
  5~9次  → 分数 ×2，轻震动反馈
  10次+  → 分数 ×3，特效 + 节奏震动
  
断掉条件：
  - 点到雷（唯一断Combo方式）
```
- 目的：让玩家进入"节奏流"状态
- **设计原则**：不用停顿时间断Combo。扫雷需要思考，强制快速点击会逼玩家盲按，失去策略趣味。让思考型和速度型玩家都能享受Combo。

#### 机制3：Combo赌注（风险收益）
- 当Combo达到5+时，屏幕轻提示"赌注"按钮
- 点击"赌注"后，下一次正确点击分数翻倍
- 如果下一次踩雷，Combo直接归零（不只是-1）
- **决策点在Combo上而非棋盘上**，不破坏棋盘信息的纯净性
- 触发多巴胺决策兴奋："我连击7了，赌不赌？"

#### 机制4：容错机制（关键创新）
- 玩家有 **3点生命**
- 踩雷：扣1血 + 局部爆炸动画 + Combo中断
- **不直接结束游戏** → 降低挫败感，提高留存

#### 机制5：迷雾区域（核心差异化）
- 棋盘中部分格子翻开后，数字**延迟2秒才显示**，之前显示"?"
- 数字本身始终是真实的，只是来得晚 → 保留紧张感但**不破坏信任**
- 难度分级控制：
  - 新手模式：0% 迷雾格子
  - 进阶模式：10% 格子有迷雾
  - 高阶模式：20% 格子有迷雾
- 玩家必须决策：等迷雾消散再行动（安全但耗时间），还是凭直觉继续点击（快但有风险）
- 与60秒倒计时形成天然张力

#### 机制6：单局内难度递增（节奏曲线）
- **前20秒（建立期）**：雷密度低，让玩家建立节奏和信心，大面积连锁展开
- **20~45秒（加速期）**：雷密度正常，迷雾格子开始出现
- **最后15秒（高压期）**：雷密度提高，迷雾比例增加
- 每局都有从"轻松→紧张→高潮"的节奏曲线，类似音游从慢到快

#### 技能系统（后期扩展，MVP不做）
> MVP阶段不实现技能系统。技能会打断"点击→反馈"的节奏流，增加决策成本。
> 如果后期加入，做成**被动触发**而非主动使用：
> - 连击10次 → 自动扫描周围一圈（不需要玩家手动操作）
> - 连击15次 → 自动标记一个雷
> - 不打断节奏，不增加认知负担

## 四、社交系统 — 同图比分 + 每日挑战

### 4.1 核心模式：同图比分挑战（推荐主模式）

**流程（极简自然，不需要额外操作）：**
1. 玩家A 正常玩一局
2. 结算时系统自动生成"挑战链接"（含地图种子 + A的分数）
3. A一键分享到微信/朋友圈
4. 好友B 点击链接 → 玩同一张地图
5. 比分数 → 双方都能看到对比结果

**为什么比"埋雷"更好：**
- A不需要额外操作（埋雷本身不好玩，等B玩的周期太长）
- 社交传播更自然（类似"跳一跳"分享挑战）
- 开发成本极低（只需要种子同步）

### 4.2 每日种子地图（核心留存机制）
- 每天全球同一张地图（固定种子），所有人玩同一局
- 每日排行榜（今日最高分 / 最长连击 / 最快通关）
- **Wordle模式验证过的留存逻辑**：每天一次机会，制造"明天再来"的期待
- 开发成本极低，留存效果极好

### 4.3 社交传播钩子（决定能不能出圈）

#### 可分享的结算画面
- 结算界面设计成适合截图/分享的样式
- 展示关键数据："60秒 | 连击23次 | 1840分 | 超越92%玩家"
- 视觉风格与游戏一致（极简科技风），不要花哨分享卡片

#### 精彩回放（3~5秒）
- 自动录制本局**最大连锁展开的瞬间**
- 可一键分享为短视频/GIF
- 在抖音/朋友圈的传播效率远高于静态排行榜

### 4.4 实时对战（后期扩展）
- 两人同时扫同一张地图
- 可看到对方进度条和失误次数（制造压力）
- 连击10次 → 给对方增加迷雾区域
- 胜负：**分数 + 时间混合制**

## 五、游戏循环

```
打开游戏（< 1秒）
     ↓
直接开始（无菜单，一键即玩）
     ↓
建立期（前20秒）：低密度雷区，大面积连锁展开，建立节奏和信心
     ↓
加速期（20~45秒）：雷密度正常，迷雾格子出现，进入Combo流状态
     ↓
高压期（最后15秒）：雷密度提高，迷雾增加，Combo赌注决策
     ↓
结算触发（三选一）：3条命用完 / 清完棋盘 / 60秒到
     ↓
展示分数 + 每日排行 + 分享挑战 + "再来一局"
     ↓
0.5秒重开 → 循环
```

**结算触发说明**：60秒是上限而非必须用完。玩得好可以45秒清完，运气差可能30秒就死。

## 六、反馈系统（三位一体，决定成败）

### 6.1 视觉反馈

| 场景 | 动效 | 时间要求 |
|------|------|----------|
| 普通点击 | 格子缩放(1→0.9→1) + 透明度渐变 | < 100ms |
| 连锁展开 | 波纹式延迟翻开，每格30~50ms | 总计 < 1s |
| Combo跳字 | 小字轻浮动 + 快速消失（不要大字飞屏） | 300ms |
| 踩雷 | 局部红闪 + 轻微屏幕震动 | 200ms |
| 高Combo | 边缘微光晕 + 颜色升温 | 持续 |

**核心原则：**
- 静态时极简（安静），操作时爆发（爽）
- 不要全屏爆炸、不要长时间停顿
- 连锁展开的**延迟传播 = 爽感来源**

### 6.2 触感反馈（手机必做）

| 操作 | 震动 |
|------|------|
| 普通点击 | 无 |
| 连击中（5+） | 轻震（10ms） |
| 踩雷 | 强震（50ms） |
| 高Combo（10+） | 节奏震动（短-短-长） |

- 使用原生插件实现分级震动（Unity Handheld.Vibrate 太粗糙）

### 6.3 音效反馈

| 场景 | 音效设计 |
|------|----------|
| 点击 | 清脆电子音/玻璃音 |
| 连击1~5 | 轻音，音调递增 |
| 连击5~10 | 加速，节奏感增强 |
| 连击10+ | 爆发音，明显高潮感 |
| 踩雷 | 低频爆炸音（冲击感） |
| 背景 | **不用音乐**，用轻电子环境音 |

**音效 = 节奏引导器，游戏成败50%在音效**

### 8.3 棋盘生成算法要点

1. **安全区保证**：首次点击的格子及周围一圈（3×3）绝对无雷
2. **种子驱动**：使用种子(Seed)初始化随机数，相同种子 = 相同地图（同图比分 + 每日挑战的基础）
3. **随机布雷**：Fisher-Yates 打乱后取前 N 个位置
4. **计算邻接数**：标准8方向统计
5. **难度递增布雷**：棋盘分为3个区域（对应建立期/加速期/高压期），雷密度递增
6. **迷雾注入**：按难度阶段百分比随机选取格子标记为迷雾，FogRevealTime = 2秒

### 8.4 连锁展开算法

```
BFS 从点击格开始：
  - 如果当前格 AdjacentMines == 0：翻开，并将8邻居加入队列
  - 如果当前格 AdjacentMines > 0：翻开，不扩散
  - 记录翻开顺序 + 距离 → 传给动画系统做延迟播放
```

### 8.5 关键动效实现（DOTween）

```csharp

```

### 9.1 推荐路线

### 9.2 同步策略

- 同步**操作序列**而不是结果（确定性重放，减少带宽）
- 地图种子同步（保证双方地图一致）
- 本地立即响应 + 服务器校验（点击型游戏延迟敏感）

## 十二、关键风险与避坑

| 风险 | 对策 |
|------|------|
| 动画手感差 → 游戏直接废 | 第二阶段重点打磨，用DOTween精调曲线 |
| 规则太复杂 → 新手流失 | 每局只变1~2个变量，分段解锁 |
| 节奏断裂 → 无聊 | 每个操作必须有即时反馈，Combo只被踩雷打断 |
| 数字不可信 → 信任崩塌 | 所有数字始终真实，迷雾只延迟不篡改 |
| 变成"思考游戏" → 节奏崩溃 | 时间限制 + 难度递增制造节奏曲线 |
| 加太多功能 → 认知过载 | 遵循"Less is More"，MVP不做技能系统 |
| 社交链路太重 → 无人分享 | 同图比分自动生成，一键分享，零额外操作 |

---

## 十三、一句话总结

> **不要把扫雷做得更"聪明"，而要把它做得更"爽"。**
> **真正让人上瘾的不是"更难"，而是"更紧张 + 更不确定 + 更快反馈"。**
